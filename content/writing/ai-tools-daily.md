I build automations for clients. I use AI constantly. But the tools I actually have running in my own workflow aren't the ones people write about.

Here's what I actually use, and why.

---

**Granola**

I stopped taking notes on calls. Granola records every meeting, transcribes it, and sends me a summary on my phone when it's done. I open it after the call, scan it, done.

Before this, I was half-present in every meeting — trying to listen and write at the same time. Now I'm fully there.

The summary isn't perfect. It misses tone, gets things slightly wrong sometimes. But it's 80% of what I need and takes zero effort. That's the trade I'll take every time.

---

**Claude → Markdown → Obsidian**

This one changed how I think.

Every time I work through something with Claude — a decision, a deep dive, a strategy session — I ask it to output the key information as a markdown file. That file goes straight into my Obsidian vault.

So instead of conversations disappearing, they become knowledge. They get connected to other things I've learned, other things I've written. They compound.

I started doing this almost by accident. Now it's the core of how I actually retain anything.

---

**n8n for everything I don't want to do twice**

n8n is my unfair advantage. Not because it's the most impressive tool — it's not. Because I've been using it long enough that I can build automations fast, and automation is leverage.

Anything I find myself doing more than twice gets automated. Client onboarding, status updates, form data routing. It runs in the background and I don't think about it.

---

The tools I use aren't the most hyped ones. They're the ones that reduce friction on things I actually need to do. Granola removes friction from calls. The Claude pipeline removes friction from learning. n8n removes friction from repetitive work.

That's the filter I use now. If a tool makes me think harder about using it than about the work itself, it's the wrong tool.
